Security checklist

Operational security that scales with partners

Use this checklist when briefing IT, legal, or procurement teams on how MIROZA protects content systems and customer data.

Controls

  • Role-based access with just-in-time elevation for editorial tooling.
  • All data in transit uses TLS 1.3; at rest encryption keys rotate every 90 days.
  • SSO + hardware-backed MFA available for newsroom partners.

Monitoring

  • Error budgets tracked by service; regressions alert the on-call desk.
  • Automated dependency scans run nightly with SBOM exports.
  • Quarterly tabletop exercises drill outage, data-loss, and abuse scenarios.

Assurance

  • SOC 2 binder updated in lockstep with policy and infrastructure changes.
  • Pen tests scheduled twice per year with remediation tracked in Jira.
  • Security contact: security@miroza.online (PGP available on request).

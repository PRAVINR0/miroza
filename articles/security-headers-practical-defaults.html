Security Headers: Practical Defaults

By MIROZA Security — Apr 15, 2025

Security headers are low-cost, high-impact controls. Start with a conservative set of defaults and iterate using report-only modes when possible.

HSTS

Set Strict-Transport-Security with a long max-age once HTTPS is stable. Example: Strict-Transport-Security: max-age=63072000; includeSubDomains; preload.

Content-Security-Policy

Begin with a restrictive CSP in report-only to discover compatibility issues. Block inline scripts and rely on nonces or hashes for trusted inline code.

Other headers

Send X-Content-Type-Options: nosniff, Referrer-Policy: no-referrer-when-downgrade (or stricter), and Permissions-Policy to limit powerful browser features.

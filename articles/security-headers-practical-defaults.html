Security Headers: Practical Defaults

Security headers are a simple and effective way to harden web applications against common threats. This guide lists recommended default headers and why they matter.

Strict-Transport-Security (HSTS)

Ensure HSTS is set to force HTTPS and protect against protocol downgrade attacks. Use a long max-age and includeSubDomains when confident.

Content-Security-Policy (CSP)

Apply a CSP that limits allowed sources for scripts, styles and images. Start with a report-only policy to surface violations before enforcing.

X-Content-Type-Options

Send X-Content-Type-Options: nosniff to prevent MIME type sniffing.
